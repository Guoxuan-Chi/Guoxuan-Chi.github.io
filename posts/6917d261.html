题目链接：无重复字符的最长子串

题目描述

给定一个字符串，找出不含有重复字符的最长子串的长度。

示例

输入："abcabcbb"

输出：3

解释：无重复字符的最长子串是 “abc”，其长度为 3。

输入："bbbbb"

输出：1

解释：无重复字符的最长子串是 “b”，其长度为 1。

输入："pwwkew"

输出：3

解释：无重复字符的最长子串是 “wke”，其长度为 3。请注意，答案必须是一个子串，”pwke” 是一个子序列而不是子串。

题目分析

  1. 暴力法：

    • 从下标i = 0开始遍历序列,并对于任意j > i, 分析区间[i, j]`内的字符是否构成一个无重复字符的子串。

    • 若构成无重复字符的子串，则记录该子串长度count，并与目前最长无重复子串长度max进行比较，若满足max < count，则更新max。

    • 若不构成无重复字符的子串，则可以认为以i为开始不可能存在更长的无重复字符子串了，由此执行i++，开始下一轮循环。

      通过分析可知，起始下标i构成一次遍历，终止下标j构成一次循环遍历。而使用集合的方法查[i, j]是否构成一个无重复序列子串也需要线性时间复杂度。因此可大致认为该方法的时间复杂度为$O(n^3)$。

  2. 滑动窗口法：

    通过分析暴力法的时间复杂度，我们可以发现关于i, j 的二次循环是存在冗余的。也就是说，不同的[i, j]很可能存在重叠的子区间。因此对于每一个[i, j]区间执行 IsUnique()操作是完全没有必要的。我们可以采用滑动窗口的思路，尽可能避免重复构造存在重叠的 hash set 并执行查找。

    • 将下标i, j赋初始值0，根据下文中的执行内容可知，i, j始终能保持 j >= i 的关系。

    • 若 s[j] 存在于 hash set 中，则认为[i, j]区间存在重复的元素，即以 i为开始不可能存在更长的无重复字符子串了。此时将 hash set 中的 s[i]元素删除，并执行 i++，则可以认为区间左边界向右滑动了。

    • 若s[j] 不存在于 hash set 中，则认为[i, j]区间不存在重复元素，将 s[j] 加入hash set 中，并尝试扩展右边界，执行 j++操作。鉴于无重复字符的子串变长了，我们需要比较 max与 j - i 的大小（由于已经执行了 j++，因此并不是 j - i + 1），并在必要时更新。

    考虑最多可能执行$2n$次操作（$n$次 i 右移，$n$次 j 右移），该算法的时间复杂度可记为为$O(2n)$ ≈ $O(n)$。

  3. 优化的滑动窗口法：

    分析滑动窗口法，我们发现当 s[j] 发生重复时，i 仅仅向右移动一次，而实际上发生冲突的元素可能处在原[i, j]中的任意位置，我们更希望若 s[j]与 s[k] (i <= k <= j )发生重复时，左边界i 能够直接移动到 k + 1 的位置。但使用 set数据结构无法定位到重复元素，因此我们将使用map <char, int>结构或是定义数组int[128]完成这项工作（ASCII码共有128种），通过 char 变量 s[j] 定位出下标 k + 1并赋值给 i。这样算法复杂度可被降低为$O(n)$，相比一般的滑动窗口法缩短近一半。
