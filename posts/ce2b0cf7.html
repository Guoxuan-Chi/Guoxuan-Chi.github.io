使用Visual Studio Code与Mingw-w64搭建轻量级C++ IDE

发表于 2019-01-24 | 分类于 Tutorial
字数统计: 2.3k | 阅读时长 ≈ 9

引言

对于大多数编程入门者与研究人员来讲，使用一个专门化的IDE可能并不是一个非常好的选择。一方面，大多数高集成度的IDE（如Visual Studio）仅仅支持有限几种编程语言的开发；另一方面，这类IDE往往体量较大，相对臃肿，更适合用于大型项目的开发而非小型的Demo、仿真或者学习使用。

  1. 为何选择了Visual Studio Code + Mingw-w64

    • Visual Studio Code是微软公司开发的一款代码编辑器，支持包括语法高亮在内的各种功能，同时内含各类语言插件支持（包括但不限于C/C++，C#，Java，Python，Markdown），被认为是取代Vim、Sublime、Atom等的新一代编辑器。

    • 相比于MinGW-w64，MinGW对于大多数人来说更加熟悉一些。MinGW是Minimalist GNU for Windows的缩写，它是一个可自由使用和自由发布的Windows特定头文件和使用GNU工具集导入库的集合，允许你在GNU/Linux和Windows平台生成本地的Windows程序而不需要第三方C运行时（C Runtime）库。然而，MinGW似乎很久没有更新了，而且从名字就能看出来，MinGW-w64对64位环境的支持更好，因此我们选择使用后者。

  2. Visual Studio Code与MinGW-w64之间的关系

    • 如上文所述，有别于Visual Studio这类集成开发环境，Visual Studio Code仅仅提供代码的编辑功能以及围绕编辑进行的辅助功能，但并不提供编译或者解释语言的能力。

    • MinGW-w64中包含几乎全部的C/C++库文件以及GCC、G++、GDB等编译调试器。可以将C/C++源码编译链接为可执行程序，并提供调试功能。MinGW-w64本身不提供编辑代码的功能：开发者们可以通过命令行的方式调用其中组件，但是这种风格显然不是大多数开发者能够接受的。

完成Visual Studio Code的插件配置

  1. 安装中文支持插件（可选）

    鉴于本文大部分受众的母语为中文，将开发环境的语言设置为中文应当能够在很大程度上提高用户的使用与开发效率。对于已进行过此步骤或英文水平较高的用户，可以跳过本步骤。

    如上图所示，单击左侧栏最下方图标，即可进入扩展插件栏。在上方搜索框中输入关键字“Chinese”，选择中文简体扩展包安装即可。安装后，在locale.json中添加”locale”: “zh-cn”，即可载入中文（简体）语言包。要修改 locale.json，你可以同时按下Ctrl+Shift+P打开命令面板，之后输入 “config” 筛选可用命令列表，最后选择配置语言命令。完成修改后可能需要手动重新启动Visual Studio Code方可生效。

  2. 安装C++语言支持插件

    单击左侧栏最下方图标，即可进入扩展插件栏。在上方搜索框中输入关键字“C++”，选择星标的C/C++扩展包安装即可。

  3. 安装Code Runner插件支持

    在扩展插件栏搜索框搜索关键字“Code Runner”，找到扩展包并完成安装即可。全部扩展包安装完成后，可能需要重新启动Visual Studio Code方可生效。
